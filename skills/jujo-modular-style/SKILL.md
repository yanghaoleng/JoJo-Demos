# JOJO 模块化视觉风格 Skill

调用本 skill 时，**把以下规范视为硬性约束**。所有模块化视觉/工程产出都要主动对齐。

> 源文档：`design/JOJO_MODULAR_VISUAL_STYLE.md`
> 色板源：`叫叫颜色色板.tokens.json`
> Figma 节点：「✏️标准题主题化」`2287:18916` / 「🎮 JOJO 游戏 / 常用动效」`9461:14060`

## 一、设计目标

叫叫模块化视觉风格用于**把通用题目放进不同学习情绪的皮肤里**：
- 题目结构保持稳定（题干 + 材料 + 选项 + 提交）
- 皮肤只负责调节氛围（背景、强调色、按钮）
- 当前落地版本聚焦**选择题**，避免用连线题增加理解成本

## 二、视觉原则（10 条铁律）

1. **题干区 + 作答区 = 纯白底**：保证阅读、选项识别、批改反馈不被主题色干扰。
2. **题干区不重复题型说明**：喇叭左侧原位，主问题占满右侧信息位。
3. **题目内容不额外加圆角描边容器**：材料、公式、提示保持开放排版，避免"卡片套卡片"。
4. **大面积背景避免渐变**：用纯色背景 + 局部强调色 + 按钮 + 状态反馈表达氛围。
5. **选项按钮圆角保持克制**：仅轻微圆角，不用大胶囊圆角，避免幼稚和松散。
6. **选项区不横向铺满右侧**：选项栈保持较短宽度，把视觉重心让给题干和材料。
7. **选项按钮可有浅色硬边投影，不用模糊阴影**：投影只表达轻微抬起感。
8. **文字按钮圆角与选项按钮一致**：提交、主题切换等都不使用大胶囊。
9. **提交按钮必须比选项按钮更大更粗**：箭头使用无尾巴符号（→ 而非 ↪）。
10. **非黄色调主题下的提交按钮用白色文字**：确保蓝色/紫色主题对比度清晰。

| `--jojo-radius-small` | 24px | 工具按钮、弹层控件 |
| `--jojo-radius-large` | 52px | 背景板大圆角 |
| `--jojo-radius-max` | 92px | 答题页外层圆角 |
| `--jojo-space-page` | 20px | 基础页边距 |
| `--jojo-space-small` | 16px | 小间距 |
| `--jojo-space-medium` | 28px | 选项栈间距 |
| `--jojo-space-large` | 40px | 题干、选项内边距 |

字号阶梯：
- 主问题题干：**44–48px / 1.16**
- 选项文字：**40px / 1.35**
- 材料标题：**36px / 1.25**
- 计数器：**30px / 1.5（圆体风格）**
- 全局 `letter-spacing: 0`

## 六、常用动效（6 套）

| 动效 | 触发 | 参数 | 落点 |
| --- | --- | --- | --- |
| `ani-btnpress-scal` | While pressing | 中心点放大 1.5x；600ms；弹簧 Quick `300/20/1` | 顶部返回、工具按钮 |
| `ani-btnpress-down` | While pressing | Y +10px；600ms；弹簧 Quick | 选项行、提交、主题切换 |
| `ani-tip-repeat` | 自动循环 | Y 上下 ±10px；300ms in/out | **答题页不启用** |
| `ani-seal-down` | 自动播放 | 2x → 1x；透明度 0 → 1；600ms | 选中答案的成功勾 |
| `ani-toast-up` | 自动播放 | Y -50px 入场/离场；400ms；停留 2000ms | 提交成功的 toast |
| `ani-bounds` | 元素被强调 | Y ±12px / ±6px；间隔 1200ms | **答题页不启用** |

实现约束：
- CSS 类名沉淀为同名 `ani-*`
- `prefers-reduced-motion: reduce` 下保留最终态、停用循环和位移

## 七、组件结构

| 类 | 职责 |
| --- | --- |
| `.jojo-adaptive-stage` | 视口居中、根节点缩放 |
| `.jojo-adaptive-root` | JS 写入 `--jojo-root-width/height`、`--jojo-stage-scale`（含预览比例） |
| `.jojo-quiz-screen` | 答题页皮肤容器（仍按 1756×810 的 2x 设计） |
| `.jojo-board` | 白色答题板和主题背景板 |
| `.jojo-question` | 题干区（64px 语音按钮 + 主问题题干） |
| `.jojo-choice-region` | 选择题内容区：左侧材料卡 480×480，右侧三项选项栈 |
| `.jojo-choice-option` | 单项选择行（664×122）+ 语音按钮 100×122 |
| `.jojo-submit` | 右下角提交按钮（未选棕禁用态 / 选择后主题强调色） |

## 八、适配 Scale

| 模式 | 2x 设计 | Unity 逻辑 | 网页预览 | 内容缩放 |
| --- | --- | --- | --- | --- |
| 手机横屏 | 1756 × 810 | 878 × 405 | 0.5 | 1 |
| 平板模拟 | 1440 × 1080 | 720 × 540 | 0.5 | 0.72 |

自动规则：视口比例 ≤ 1.55 或最短边 ≥ 760 → 平板模拟；否则 → 手机横屏。

## 九、违规检查清单

- [ ] 题干/作答区是否保持纯白底？
- [ ] 是否在题目内容中加了"卡片套卡片"？
- [ ] 大面积背景是否避开了渐变？
- [ ] 选项按钮圆角是否为 12px（克制）？
- [ ] 选项栈是否保持较短宽度？
- [ ] 选项按钮投影是否硬边（非模糊）？
- [ ] 提交按钮是否比选项按钮更大更粗？
- [ ] 非黄色调主题下提交按钮是否用白色文字？
- [ ] 答题页是否启用了被禁的循环动效（`ani-tip-repeat` / `ani-bounds`）？
- [ ] 是否给了 `prefers-reduced-motion` 降级？

任一项不通过，先改完再交付。
