# JOJO UI Design MD

调用本 skill 时，**把以下规范视为硬性约束**，任何设计/工程产出都要主动对齐，写代码或写文档时遇到冲突项要按本规范执行。

> 完整色板 JSON：`叫叫颜色色板.tokens.json`（JUDS 工程内）。
> 规范版本：1.3

## 一、品牌定位与设计原则

### 1.1 品牌定位

叫叫 = 4–12 岁儿童成长数字内容平台（阅读 / 英语 / 思维 / 美育 / 小作家）。

| 关键词 | 设计含义 |
| --- | --- |
| 温暖 | 明亮、亲近、鼓励式表达，避免压迫感 |
| 童趣 | 有角色、有故事、有反馈，但不低幼化 |
| 清晰 | 信息层级明确，阅读负担低，核心行动明确 |
| 专业 | 内容可信、承诺克制、效果表达具体 |
| 包容 | 孩子探索感 + 家长信任感 |

### 1.2 设计原则（4 条不可妥协）

1. **有趣但不低幼**：柔和、明亮、有角色；不堆叠游戏化和营销化元素。
2. **专业但不冰冷**：清晰信息架构 + 真实效果表达，不用冷色调和复杂术语伪装专业。
3. **先反馈，再执行动作**：重要跳转/下载/提交/外链用轻量二次确认或明确 pressed 态。
4. **反馈分层**：一次反馈至少两层——**背景（状态）+ 内容（意图）+ 物理感（触感）**。

## 二、色彩 Token（只允许使用以下值）

### 2.1 中性色

| Token | 值 | 用途 |
| --- | --- | --- |
| `gray-1` | `#FAFAFA` | 极轻背景 |
| `gray-2` | `#F5F4F4` | 描边、弱分割 |
| `gray-3` | `#E0DFDF` | 控件描边、禁用背景 |
| `gray-4` | `#B2B2B2` | 弱文字、辅助 icon |
| `gray-5` | `#666666` | 正文辅助信息 |
| `gray-6` | `#404040` | 主文字 |
| `ink` | `#2F2A1F` | 品牌官网标题、强叙事标题 |
| `white` | `#FFFFFF` | 页面、内容承载 |

### 2.2 品牌主色

| 名称 | Token | 值 |
| --- | --- | --- |
| 叫叫黄 | `yellow-4` | `#FCDA00` |
| 绿豆绿 | `green-4` | `#78C562` |
| 阳光橙 | `orange-4` | `#FF9045` |
| 樱花粉 | `pink-4` | `#FF7A99` |
| 天空蓝 | `blue-4` | `#5AC8FA` |
| 葡萄紫 | `purple-4` | `#AE84E3` |
| 专业蓝 | `pro-blue-4` | `#528BFF` |

每个主色都有 1–6 的明度阶梯（如 `yellow-1 #FCF9DE` / `yellow-6 #544300`）。**禁止跨主色混搭出新的"自定义色"**。

## 三、字体系统

- 中文：标题 `Mohr Rounded Bold`（圆润、童趣），正文 `PingFang SC`
- 英文：标题 `Mohr Rounded`，正文 `-apple-system` / `BlinkMacSystemFont` / `Segoe UI`
- 等宽：`SF Mono` / `Menlo` / `Consolas`（仅用于代码片段、npx 命令、token 值）

字号阶梯：12 / 14 / 16 / 19 / 24 / 32 / 48
行高阶梯：1.22（标题）/ 1.45（正文）
字间距：`letter-spacing: 0`

## 四、形状与间距

| Token | 值 | 用途 |
| --- | --- | --- |
| `--radius-card` | 22px | 卡片、容器 |
| `--radius-pill` | 999px | 主按钮、胶囊 |
| `--radius-sheet` | 16px | 弹层、底部弹窗 |
| `--radius-chip` | 10px | chip、小标签 |

**禁止**新增第三种圆角风格。每个圆角值都对应一个使用场景，不要在按钮上用 22px 也不要给卡片用 999px。

间距：4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 八档。仅在跨档场景下使用 6 / 10 / 14 这种"过渡值"，并在注释中说明。

## 五、动效

主引擎：GSAP。

| 场景 | 时长 | 缓动 |
| --- | --- | --- |
| 入场 | 0.6 – 0.8s | `easeOutBack`（轻弹簧） |
| 退场 | 0.3 – 0.4s | `power2.out` |
| 反馈闪烁 | 0.18s | `linear` |
| 列表错位入场 | 0.5s + 0.05s 步进 | `easeOutBack` |

约束：
- 默认尊重 `prefers-reduced-motion: reduce`，停用循环和位移动画
- **禁止**循环呼吸、上下浮动等持续动效干扰阅读
- 弹簧参数统一：`stiffness 300 / damping 20 / mass 1`

## 六、组件状态模型

每个交互组件至少包含 6 态：`default / hover / pressed / focus / disabled / loading`。

- **default**：背景 `gray-2` 弱底 / 文字 `gray-6`
- **hover**：背景提亮 1 档 / 阴影出现（无模糊）
- **pressed**：缩放 `0.96` / 阴影消失
- **focus**：2px 描边，使用 `yellow-4`
- **disabled**：`gray-3` 描边 + `gray-4` 文字
- **loading**：保留形状，旋转 icon，禁止用"按钮宽度变化"做 loading

## 七、可访问性

- 颜色对比度：正文 ≥ 4.5:1，大字 ≥ 3:1
- 焦点环永远可见（不依赖 `:hover` 模拟）
- 所有 icon 按钮必须有 `aria-label`
- 表单错误用文字+颜色双通道表达，不只用红色

## 八、违规检查清单

完成任何 UI 产出前自检：

- [ ] 是否只用本规范里的色值？
- [ ] 圆角是否只用了 4 个 token 值之一？
- [ ] 字号、行高、字间距是否落在阶梯内？
- [ ] 动效是否在 0.3 – 0.8s 范围内且符合缓动规范？
- [ ] 每个组件是否覆盖 6 态？
- [ ] 是否给了 `prefers-reduced-motion` 降级？
- [ ] icon 按钮是否有 `aria-label`？

任一项不通过，先改完再交付。
